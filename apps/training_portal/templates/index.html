EECampEdu — AI PC Training Portal

Train, quantize, and flash gesture models from your browser.

1a.Import dataset (.zip)

Upload one zip with exactly six class folders, any names (e.g. n1/*.jpg … n6/*.jpg). It auto-detects the layout: if the zip already has train/ and validation/, both import as-is; otherwise the six folders are auto-split into train/validation. The class order is saved to class_mapping.json and used everywhere.

1b.Map classes to robot actions

Each trained class (model output index) drives one robot-arm action. The ESP firmware uses only the index; this mapping is applied on the student PC control app. Saved to class_mapping.json. Classes: —

2.Quantize for ESP32-S3

Converts a trained .keras model into a deployable int8 TFLite + quantization report. The result appears in Generated artifacts below and in the Flash model tab.

3.Flash model partition (.tflite)

Flash only the model into its flash partition — the main board firmware is untouched. Plug the ESP32-S3 into your PC, use Chrome or Edge; the browser asks you to pick the board's serial port. No install, no offsets.

This page is served over HTTP, so Chrome/Edge hide serial flashing. Allow this one site once (per PC), then reload:

  1. Copy this address:
  2. In a new tab open chrome://flags/#unsafely-treat-insecure-origin-as-secure
    (Microsoft Edge: edge://flags/#unsafely-treat-insecure-origin-as-secure) — type it into the address bar.
  3. Paste the address into the box, set the dropdown to Enabled, then click Relaunch.
  4. Return here, reload, and click Connect ESP32-S3.

If your school browser blocks this flag, ask your instructor to serve the portal over HTTPS.

Click “Connect ESP32-S3” to begin.

4.Flash main board firmware

Flashes the full ESP32-S3 main board firmware (camera + USB + TFLite Micro inference) built with ESP-IDF. This is separate from flashing a model — do this once to set up a board, or to update the firmware. Offsets come from the ESP-IDF build metadata (flasher_args.json); students never type them.
